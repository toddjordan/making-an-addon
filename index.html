json-schema ftw

A Component

  • Drop it right into your component hierarchy
  • Provide it with ember data models, closure actions
  • Then just load your schema, that you store wherever (ember data, js payload, etc)

A Framework for Reuse

  • Goal is to take all code out of the json schema
  • Use Ember Magic: Blueprints, and the Ember Resolver

Upgrading

  • You can keep addons current and still be backwards compatible, as long as you are careful about using new APIs
    • Sometimes you can use a polyfill addon to be backwards compatible (ex. getOwner)
    • Test compatibility as part of your build with ember-try.
  • Follow the steps at https://github.com/ember-cli/ember-cli/releases - Addon developers section. (Trick with the diff)

Deploying

4 Magic commands

  • npm version your.version
  • git push origin master
  • git push origin --tags
  • npm publish

Be careful, publish can be undone, but its a pain. Make sure you are on the right branch and that you have your master up to date!

ember-cli-github-pages for live documentation!

  • git checkout master
  • ember github-pages:commit --message "Initial gh-pages release"
